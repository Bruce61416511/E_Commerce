传统选品靠经验拍脑袋，AI 把五步串成流水线：看信号 → 找需求 → 出概念 → 做筛选 → 定价格

产品研发方法论
五步产品概念生成法，把传统的"拍脑袋选品"变成"流水线造品"。第一步看信号：不急着想卖什么，先看市场在发生什么——刷TikTok热点、盯Amazon飙升榜、看1688上哪些品类询盘在涨，目标是找到正在变热的方向，而不是已经红海的赛道。第二步找需求：信号只是表象，得翻译成用户需求。比如露营灯搜索量涨了300%，背后是谁、在什么场景、想解决什么问题——是亲子露营需要安全柔光，还是徒步客要超轻续航？这一步把数据变成用户画像。第三步出概念：需求明确了，AI批量发散产品方向，一个需求卡片扔进去，吐出来50个概念，每个都带品类定位、目标人群、核心卖点、预估价格带。人做判断，AI做发散。第四步做筛选：50个概念不能全做，用四个硬指标过滤——搜索量有没有人搜、竞争度对手强不强、毛利空间赚不赚钱、供应链1688能不能做。每个概念打分排名，留下TOP 5。第五步定价格：把TOP 5里最靠谱的那个落地成产品Brief，包含目标用户画像、核心功能规格、定价策略、差异化定位、上市路线图。这个Brief直接对接后面的竞品拆解和选品决策环节。

# 产品概念生成 · 全流程 Prompt

## 角色
你是电商产品策略顾问。基于多源数据，完成从市场信号捕捉、用户需求挖掘到跨界概念生成的全流程分析，最终输出可落地的产品开发建议。

---

## 第一步：看信号 — 多源数据采集 & 趋势发现

### 数据采集
使用 🔧 八爪鱼 / Apify / 蝉妈妈 采集以下数据：
- 热搜数据：各平台搜索趋势（Amazon搜索词、1688询盘热词、Google Trends）
- 销量数据：各平台热销榜（Amazon Best Sellers、TikTok Shop爆款、Shopee热销）
- 社媒数据：社交消费趋势（TikTok热门话题、小红书种草笔记、Instagram趋势标签）

### AI 分析
用 🧠 GPT-4o 完成：
1. 多源爬取 + 清洗标准化：将不同来源的数据统一为结构化格式，去除重复和噪声
2. 识别上升品类 + 增速排序：找出搜索量、销量、讨论量同步增长的品类，按增速降序排列 TOP 10
3. 判断趋势真假：区分短期炒作（突然爆量后回落）和长期趋势（稳定增长 3 个月以上），标注每个品类的趋势可信度

### 输出要求
- 搜索量趋势（月环比增长率）
- 转化率（搜索到购买的转化比例）
- 评论热度（评论数增速 + 情感倾向）
- 增速排名 TOP 10 + 趋势真假判断
- 3 个最值得切入的品类，每个附 100 字入场理由

---

## 第二步：找需求 — 差评痛点挖掘

### 数据采集
使用 🔧 八爪鱼 / 魔镜市场情报 采集：
- 用户评论：目标品类 TOP 20 商品的 1-2 星差评（覆盖 Amazon / 淘宝 / 京东 / Shopee）
- 口碑数据：社交媒体用户反馈、问答平台讨论、客服投诉记录

### AI 分析
用 🧠 GPT-4o 完成：
1. 提取差评高频词：按出现频次统计 TOP 20 负面关键词（如“太重”、“续航短”、“掉色”）
2. 聚类归纳痛点：将高频词归纳为 5-8 个核心需求缺口，区分功能性痛点（产品不好用）和体验性痛点（用起来不爽）
3. 排序 + 情景模拟：按影响面（提及人数 × 严重程度）排序，模拟改进后的用户满意度变化

### 输出要求
- 5-8 个核心需求缺口，每个标注频次、影响面、痛点类型
- 每个缺口给出具体的产品改进方向（100 字内）
- 痛点 × 改进方案矩阵，标注优先级（高/中/低）

---

## 第三步：出概念 — 跨界机会发现

### 数据采集
使用 🔧 八爪鱼 / Apify 采集：
- 跨品类购买数据：用户在同一店铺或同一订单中同时购买的不同品类商品
- 品类关联数据：搜索“A + B”组合词的频次、社交平台上的品类搭配讨论

### AI 分析
用 🧠 Neo4j 图数据库 + GPT-4o 完成：
1. 构建品类关联图谱：将跨品类购买数据建模为节点-边关系图，发现隐性关联
2. AI 跨界推理：识别强关联但供给不足的品类组合（如“露营 + 咖啡”、“瑜伽 + 营养补剂”）
3. 可行性验证：评估市场规模、竞争强度、供给门槛三个维度

### 输出要求
- 5 个跨界产品方向，每个标注关联强度和供给密度
- 每个方向附市场规模预估、现有竞品数量、1688 工厂匹配度
- TOP 3 推荐方向，附 200 字可行性判断

---

## 最终输出格式
使用 Markdown 表格 + 要点列表，不超过 2000 字：

| 维度 | 看信号 | 找需求 | 出概念 |
|------|--------|--------|--------|
| 核心发现 | 3 个上升品类 + 增速 | 5-8 个痛点缺口 | 3-5 个跨界方向 |
| 数据支撑 | 搜索量/转化率/热度 | 高频词/影响面/类型 | 关联强度/供给密度 |
| 行动建议 | 立即入场/观察/放弃 | 改进优先级 | 可行性判断 |

## 补充说明
- 各步骤的数据采集工具为“或”的关系，根据实际数据源选择其一即可
- 分析结果需标注置信度（高/中/低），避免过度自信
- 优先使用近 3 个月数据，趋势判断可拉长至 6 个月

# 竞品分析 Prompt 模板

```
你是一个电商竞品分析师。请基于提供的竞品价格数据，完成以下分析：

【输入数据】
- 竞品 X 近 7 天每日标价、券后价、促销标签（满减/秒杀/会员价）
- 竞品 X 近 30 天价格历史（用于对比趋势）
- 平台大促日历（618/双11/品类日等节点标注）

【分析任务】
1. 价格波动识别：标注 7 天内所有调价节点，计算调价幅度（%）、调价频率（次/天）、调价方向（涨/降）
2. 策略模式判定：将价格行为归类为以下模式之一或多选——
   · 高频调价（日均调价 ≥ 2 次，小幅浮动）
   · 阶梯降价（逐日递减，配合活动倒计时）
   · 活动前涨价（大促前 3-7 天提价，活动当天"打折"）
   · 跟随定价（与我方/行业价格走势强相关）
   · 清仓甩卖（持续降价 + 库存告急信号）
3. 营销意图推测：基于策略模式 + 促销标签 + 时间节点，推断竞品当前意图——冲销量/清库存/拉新客/利润收割/防守阻击
4. 应对建议：针对每种意图，给出我方 3 条可执行对策（调价/促销/产品差异化），标注优先级（高/中/低）

【输出格式】
使用 Markdown 表格输出，每行一项分析，含：时间 | 动作 | 模式 | 意图 | 我方对策 | 优先级
末尾附 200 字以内的结论摘要，给出「跟进 / 观望 / 反击」建议。
```

04 · 动态定价 & 案例

· 看什么：库存 / 竞品价格 / 时间 / 季节 / 促销节点 / 成本底线 / 产品阶段

· 策略：阶梯调价（日常 / 活动 / 清仓 三档）

Part · 03

供应链

AI 预测库存与需求优化 — 让库存周转飞起来

第三部分：供应链。在电商里，库存就是现金流。卖得好的品缺货是灾难，卖不动的品积压也是灾难。AI的核心价值就是让预测更准、补货更聪明。

01 · 需求预测引擎

多因素融合，让预测不再是"拍脑袋"

🧠 Temporal Fusion Transformer

输入特征：历史销量 + 促销标签/折扣深度 + 城市天气 + 节假日/大促日 + 社媒搜索指数
输出：未来30天日维度预测（含置信区间）
对比基线：Prophet 做季节性分解，TFT 做多变量复杂融合

📊 分层预测策略

爆款（日均>100件）：TFT / DeepAR 日预测
腰部门（10-100件）：LightGBM 周预测
长尾（<10件）：Croston 间歇需求模型 + 相似品聚类借用
预测-实际偏差每日回流 → 在线更新

供应链17 / 27
TFT是目前时序预测的SOTA模型——它能同时处理销量、促销、天气、节假日等多种因素，输出未来30天的预测和置信区间。分层预测的关键：爆款要日粒度精细管理，长尾商品用间歇需求模型，不要一刀切。
